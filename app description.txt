\f0\fs24 \cf0 FITNESS STUDIO TRAINING APP  \
Complete Product & Development Specification\
\
Version: 1.0  \
Purpose: Guide the full development of a cross-platform application for managing training sessions and athlete registrations in a fitness studio.\
\
This document defines the system requirements, architecture, database structure, user flows, and development roadmap.  \
The system should be implemented step-by-step according to this document.\
\
--------------------------------\
1. SYSTEM OVERVIEW\
--------------------------------\
\
The system is a cross-platform mobile application designed for a fitness studio that manages group training sessions.\
\
The application will allow:\
- Athletes to register for training sessions\
- Coaches to view participants and manage attendance\
- Managers to control the schedule and approve new athletes\
\
The system must support:\
\
1. Mobile application (iOS + Android)\
2. User authentication with username/password\
3. Role-based permissions\
4. Athlete approval process\
5. Training session registration\
6. Waiting list notifications\
7. Cancellation tracking\
8. Historical records\
9. Administrative control of the training schedule\
10. Automatic opening of registrations every Thursday at 08:00\
\
--------------------------------\

3. USER ROLES\
--------------------------------\
\
The system has three types of users.\
\
1. Athlete\
2. Coach\
3. Manager (Admin)\
\
Each role has different permissions.\
\
--------------------------------\
4. ATHLETE FUNCTIONALITY\
--------------------------------\
\
Registration\
\
An athlete must register with the following information:\
\
username  \
password  \
full name  \
phone number  \
age  \
gender  \
email  \
\
After registration:\
\
- The profile status is "pending approval".\
- The athlete cannot register for training sessions yet.\
- A manager must review and approve the profile.\
\
Manager Approval\
\
The manager will contact the athlete by phone in order to:\
\
- understand physical limitations\
- understand training goals\
- introduce the studio\
\
Only after approval can the athlete register for training sessions.\
\
Athlete Capabilities After Approval\
\
Athletes can:\
\
View all available training sessions  \
Register for sessions with available spots  \
Request notification if a session is full  \
View all sessions they registered for  \
Cancel a registration with a cancellation reason\
\
Cancellation Rule\
\
If an athlete cancels less than 12 hours before the training session, they will be charged the full session price.\
\
The system must record this event.\
\
--------------------------------\
5. COACH FUNCTIONALITY\
--------------------------------\
\
Coaches can:\
\
View all training sessions  \
View registered athletes for each session  \
View athletes waiting for notification  \
Manually add athletes to sessions\
\
Coaches cannot remove registered athletes.\
\
--------------------------------\
6. MANAGER FUNCTIONALITY\
--------------------------------\
\
Managers have all coach permissions plus:\
\
Approve new athletes  \
Remove registered athletes from sessions  \
Create training sessions  \
Edit training sessions  \
Define session capacity  \
Assign coach to session  \
Control which sessions are open for registration\
\
--------------------------------\
7. TRAINING SESSION RULES\
--------------------------------\
\
Managers define the training schedule.\
\
Each session includes:\
\
day  \
date  \
start time  \
coach  \
maximum participants  \
\
Only sessions defined by managers appear in the athlete application.\
\
--------------------------------\
8. REGISTRATION OPENING RULE\
--------------------------------\
\
Every Thursday at 08:00:\
\
Registration opens only for the following week.\
\
Example:\
\
Thursday \uc0\u8594  registration opens for next week's Sunday\'96Saturday.\
\
This must be implemented as a scheduled backend job.\
\

--------------------------------\
10. BUSINESS LOGIC RULES\
--------------------------------\
\
Only approved athletes can register for sessions.\
\
Session capacity must never be exceeded.\
\
If a session is full:\
\
Athletes can request notification.\
\
If a spot opens:\
\
The system sends a push notification.\
\
Cancellation < 12 hours:\
\
charged_full_price = true\
\
Coach permissions:\
\
add athletes\
\
Manager permissions:\
\
add athletes  \
remove athletes\

--------------------------------\

--------------------------------\

\
